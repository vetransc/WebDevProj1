Itemized breakdown of expectataions

  1. I created a website for a friend of mine for her photography business.

  2. All content is original and not lifted from other websites. There are no third-party images, they all belong to my client. The text fonts are open source.

  3. My site contains at least five separate pages, each having a meaningful purpose. One of the files is named index.html, which is my home page. I also have a about page, contact page, galleries page, info page, lifestyle page and wedding page.

  4. My site includes a header with navigation.

  5. My site includes navigation links in the header and separate navigation links in the info page.

  6. My site supportt distinct layouts for large and small screens using media queries. Multicolumn layouts on large screens are reduced to single column layouts on small screens.

  7. My site maintains a coherent style across its pages. Repeated style rules are placed in a shared stylesheet. Style rules not shared between pages are placed in separate stylesheets.

  8. my site must includes a lot of images, and their alt attributes must adequately describe the image content to users of screen readers, user with images disabled, and webcrawlers.

  9. Each page declares its favicon, the company logo.

  10. My site’s non-HTML resources areorganized. Images are placed in a directory named images. CSS files are placed in a directory named css. JavaScripts are placed in a directory named js.

  11. My site was not built using any libraries or frameworks. I did not use Bootstrap, jQuery, React, Angular, Vue, or any other third-party tools.

  12. Each page and stylesheet of your site must validate without any warnings or errors using the W3C’s Nu Validator. The content should be neatly aligned. The axes of alignment should be clearly visible. Favor left- and right-alignment over centering—as centering does not produce a strong axis of alignment.

  13. Your site must use at least two different font faces. Headings are often rendered in a sans-serif font, and paragraph copy in a serif font.

  14. Your site must include at least one form (or informal collection of input elements). The form might submit data to a server or invoke some client-side JavaScript function that alters the page. The action is not specified.

  15. Your site must include links to external sites.

  16. Your source code must be readable, using conventional and consistent formatting and meaningful names for classes and IDs.

  17. Your site must include at least one animation, powered either by CSS or JavaScript.

  18. Your site must be stored in a Git repository that you have shared with your instructor. On GitHub, GitLab, and Bitbucket, share with the user twodee.

    19. The names of your files must be entirely lowercase and contain only alphanumeric characters.

  19. Your site must have its own domain name.

  20. Your site must be served out over HTTPS on port 443. You are encouraged to use Let’s Encrypt to obtain a free certificate.

  21. Users who access your site over HTTP on port 80 must be automatically redirected to port 443.

  22. Your Git repository should show a steady flow of commits over the course of the project. Committing and pushing should happen steadily, after each work session, and not just right before the due date.
